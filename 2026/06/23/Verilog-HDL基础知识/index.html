第1章：初识Verilog

本文以 Verilog-2001 的可综合写法为主。少量来自 SystemVerilog 的常用写法（如 $clog2、$urandom、多维端口等）会单独标注，避免把语言特性和综合能力混在一起。

1.1 什么是Verilog？

Verilog HDL（硬件描述语言）是一种用于描述数字电子系统的文本语言。你可以用它来描述从简单的门电路（如与门）到复杂的微处理器（如CPU）的任何数字硬件。

核心思想：Verilog代码不是“程序”，而是“电路说明书”。当你写 a & b 时，你描述的是一个与门；当你写 q <= d 时，你描述的是一组D触发器。理解这一点，是写出高质量代码的基础。

第2章：Verilog的词法规则

2.1 标识符（Identifier）

用于命名模块、端口、信号等。

  • 规则：由字母、数字、下划线 _ 和美元符 $ 组成，但不能以数字开头。
  • 区分大小写：clk 和 Clk 是两个不同的信号。
  • 建议：采用有意义的命名，如 data_bus、address，并保持团队风格统一。

2.2 空白符与注释

  • 空白符：空格、Tab和换行在编译时被忽略，用于提高代码可读性。
  • 注释：
    • 单行注释：//，从 // 到行尾的内容均被忽略。
    • 多行注释：/* ... */，用于大段说明或临时屏蔽代码。

2.3 数字常量（Number Literals）

Verilog中有四种基本逻辑值：0, 1, x（未知）, z（高阻）。有位宽常量的常见格式为：位宽'进制数值，例如 8'hFF、4'b1010。

写法 说明 实际位宽/值
4'b1010 4位二进制数 1010
8'hFF 8位十六进制数 11111111
5'o12 5位八进制数 01010
4'b1x0z 混合逻辑值 1x0z
8'b1010 注意：位宽8，但数值只有4位 高位自动补0，结果为 00001010
'hA5 未指定位宽 默认32位，结果为 000000A5
-4'd5 负数 对 4'd5 取负；若存入4位变量，结果按补码截断为 4'b1011

第3章：数据类型详解

3.1 线网类型（Net）

wire 是线网的默认类型，用于表示物理连线。

  • 特征：不存储值，值由驱动源（如门的输出、assign语句）决定。
  • 多驱动：多个驱动源连接到同一根wire时，其值由驱动源的强度决定（通常用于三态总线）。

3.2 寄存器类型（Variable）

reg 是 Verilog 中的过程赋值变量，它表示“这个信号可以在过程块中被赋值”，不等于一定会生成真实寄存器。

  • 特征：在仿真语义中会保持当前值，直到被新的过程赋值语句改写。
  • 赋值：通常在 always 或 initial 过程块中被赋值。
  • 综合对应：可以综合为触发器（时序逻辑）、锁存器（不推荐）或纯粹的 wire/组合逻辑。

3.3 其他数据类型

  • integer：32位有符号整数，多用于for循环或仿真参数。
  • real：浮点数，用于仿真，不可综合。
  • time：64位无符号时间变量，用于保存仿真时间。

4.1 向量声明

当位宽大于1时，我们称其为向量。声明格式为 [msb:lsb]。

4.2 位选与部分位选

可以灵活地访问向量中的某一位或某一段。

4.3 数组（非压缩数组）

第5章：运算符精讲

5.1 算术运算符

+ - * / %（取模）

5.2 逻辑与按位运算符

这是最容易混淆的地方，务必区分！

  • 逻辑运算符（&& || !）：返回1位结果（True/False），常用于if条件判断。
  • 按位运算符（& | ^ ~）：对操作数的每一位进行计算，返回向量。

5.3 归约运算符

单目运算，将向量的所有位缩减为1位结果。

5.4 连接与复制运算符

  • 连接 { }：将多个信号组合成一个新向量。
  • 复制 {n{expr}}：将表达式重复 n 次。

规则：连接操作中的每一个信号位宽必须明确。

第6章：过程块（Procedural Blocks）

过程块是行为级建模的基础。Verilog中有两种过程块：initial 和 always。

6.1 initial 过程块

initial 块在仿真开始时执行一次，执行完毕后线程结束。

  • 用途：主要用于仿真测试平台（Testbench）的初始化、生成激励信号。
  • 重要：面向 ASIC 或通用可综合 RTL 时，通常不要依赖 initial 块；部分 FPGA 工具支持寄存器或 RAM 的初值初始化，但这属于器件/工具相关能力。教程中的 initial 示例按仿真用途理解。

6.2 always 过程块

always 块在仿真过程中反复执行，是描述可综合电路的主力。

  • 语法：always @(敏感列表) begin ... end
  • 敏感列表：决定过程块何时被"唤醒"执行。

三种常用的 always 写法：

写法 敏感列表 电路类型 示例
组合逻辑 @(*) 或 @(a or b or c) 纯组合电路（无时钟） 译码器、加法器、多路选择器
时序逻辑（同步） @(posedge clk) 时钟边沿触发的寄存器 计数器、移位寄存器、状态机
时序逻辑（异步复位） @(posedge clk or negedge rst_n) 带异步复位的寄存器 常见工程写法之一

关键区别：同步复位只在时钟上升沿采样复位信号；异步复位一旦复位信号有效，立即生效，不依赖时钟。

第7章：过程赋值语句（分水岭！）

这是Verilog新手最容易踩的坑，也是面试必考题：阻塞赋值（=） 与 非阻塞赋值（<=） 的区别。

7.1 阻塞赋值（Blocking Assignment，=）

  • 执行方式：顺序执行，像C语言一样。前一条语句执行完毕，才执行下一条。
  • 本质：右式计算完成后立即更新左端变量。
  • 用途：组合逻辑。

7.2 非阻塞赋值（Non-blocking Assignment，<=）

  • 执行方式：并行执行。计算所有右端表达式（使用进入always块时的旧值），然后统一更新左端变量。
  • 本质：所有非阻塞赋值会在当前仿真时间步的 NBA（Non-blocking Assignment）更新阶段统一生效，看起来就像一组触发器同时采样。
  • 用途：时序逻辑。

7.4 工程黄金法则（务必遵守！）

  1. 时序逻辑（always @(posedge clk)）：统一使用 非阻塞赋值（<=）。
  2. 组合逻辑（always @(*)）：统一使用 阻塞赋值（=）。
  3. 绝对不要混用：在同一always块中不要混合使用=和<=。

第11章：层次化设计——模块例化（Instantiation）

一个复杂的数字系统（如CPU）由无数个小模块（如加法器、寄存器、状态机）组成。模块例化就是在顶层模块中"调用"或"引用"这些子模块，就像在原理图上放置芯片一样。

11.1 模块例化的基本语法

11.2 两种端口连接方式

方式一：按名字关联（推荐！）

语法：.模块端口名(连接信号名)

  • 优点：顺序无关，可读性强，便于维护。

方式二：按位置关联（不推荐！）

语法：按照模块定义时端口列表的顺序，一一对应连接。

工程铁律：永远使用按名字关联的方式例化模块！

第17章：复位设计（Reset Design）

复位用于将电路初始化到已知状态。有两种主要的复位方式：异步复位和同步复位。

17.1 异步复位（Asynchronous Reset）

  • 特点：复位信号有效时，立即将寄存器清零，不依赖时钟。
  • 优点：电路简单，复位信号可以独立于时钟。
  • 缺点：复位释放时，如果刚好接近时钟上升沿，可能导致亚稳态（Metastability）。

17.2 同步复位（Synchronous Reset）

  • 特点：复位信号只在时钟上升沿被采样，并生效。
  • 优点：时序分析更简单，不易产生亚稳态。
  • 缺点：需要额外的复位选择逻辑，且复位信号必须持续足够长的时间被时钟采样到。

17.3 工程推荐：异步复位、同步释放

这是业界最推荐的做法，兼顾了异步复位的"立即性"和同步释放的"稳定性"。

核心思想：用一个同步器将异步复位信号"同步化"后再使用。

第18章：时钟域交叉（CDC）与亚稳态

当数据从一个时钟域传输到另一个时钟域时，如果采样时钟和数据变化不同步，就会产生亚稳态（输出处于不确定状态）。CDC设计是高级数字设计中最核心的挑战之一。

18.1 单比特信号的同步

场景：一个单比特电平信号从源时钟域进入目标时钟域。
方法：双级同步器（Two-Stage Synchronizer）。将信号连续通过两个触发器，降低亚稳态继续传播的概率。

原理：虽然第一级触发器可能进入亚稳态，但经过一个时钟周期后，它大概率已经恢复到稳定的0或1，第二级触发器再采样时，亚稳态继续向后传播的概率会显著降低。双级同步器不能从数学上“消除”亚稳态，只是把风险降到工程可接受范围内。

补充：双级同步器适合同步单比特电平信号。若是脉冲信号从快时钟域进入慢时钟域，需要脉冲展宽、toggle 同步或握手协议，否则目标时钟域可能采不到这个脉冲。

18.2 多比特信号的同步

不要直接对多比特总线的每一位分别使用单比特同步器！每个比特的采样时间和延迟可能不同，导致目标时钟域看到一个“拼出来的错误数据”。常见正确做法包括：

  • 数据跨时钟域：使用异步 FIFO。
  • 控制信号跨时钟域：使用握手协议、toggle 同步或 Gray 码指针。
  • 多比特状态/计数值跨时钟域：使用 Gray 码并配合同步器，保证每次只变化一位。

第19章：有限状态机（FSM）编码技巧

状态机是数字系统的"大脑"。选择正确的编码方式，直接影响到电路的性能和面积。

编码方式 特点 适用场景
二进制编码 使用 2'b00, 2'b01, 2'b10 状态少，组合逻辑简单，节省触发器
格雷码 相邻状态只变1位 跨时钟域指针、顺序计数状态，减少多位同时翻转
独热码（One-Hot） 每个状态独占一个触发器 高速状态机，时序优化，FPGA中最常用

FPGA中的常见经验：状态较多、速度要求较高时，独热码（One-Hot）通常更容易跑高频，因为FPGA中触发器资源相对丰富，且译码逻辑简单。状态很少或面积更敏感时，二进制编码也完全合理；实际工程中也可以让综合工具自动选择编码。

第20章：典型模块设计——同步FIFO

同步FIFO（First-In-First-Out）是数字系统中最重要的缓冲结构。它的关键在于读写指针的比较。

20.1 FIFO架构

  • 写指针（wr_ptr）：指向下一个要写入的地址。
  • 读指针（rd_ptr）：指向下一个要读出的地址。
  • 空信号（empty）：写指针和读指针完全相等，表示没有可读数据。
  • 满信号（full）：写指针和读指针的地址位相等，但额外最高位不同，表示写指针已经比读指针多绕了一圈。

区分满和空的技巧：给指针增加一个额外的最高位（MSB）作为"标志位"，当指针回绕时，标志位翻转。

  • 组合逻辑用阻塞赋值（=）。
  • 一个 always 块只处理一个信号或一组紧密相关的信号。
  • 敏感列表：时序逻辑用 posedge clk，组合逻辑用 @(*)。
