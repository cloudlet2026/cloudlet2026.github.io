Xilinx ZYNQ-7000系列AXI-DMA基础知识

发表于2026-06-24|更新于2026-07-12
|浏览量:

Manager component<—>读写事务请求的发起者，MASTER
Subordinate component<—>读写事务请求的响应者，SLAVE

AXI总线

  • AXI_FULL 高速内存寻址
  • AXI_Lite 轻量级ip核配置寄存器
  • AXI_Stream 高速流数据（只传数据不含地址）

—

 

AXI ACP和AXI_HP接口都是PL为主机，PS为从机
PL发送给PS的数据，经过AXI_HP接口后进入DDR3控制器，然后写入到DDR3中。同样，PL可通过AXI_HP接口读DDR3中的数据。 
AXI_GP接口，PS为主机PL为从机

AXI数据位宽

  • AXI(AXI4-FULL):32,64,128,256bits
    • 高性能存储器映射
  • AXI4-Lite:32bits
    • 低吞吐率存储器映射
  • AXI4-Stream(ST):高速流数据通信
  • 地址位宽：32bits
  • 配置完 HP 端口以后，ZYNQ会多出一个 AXI Slave 端口，名称为 S_AXI_HP0， 不过这些端口都是 AXI3 标准的，常用的是 AXI4 协议，添加 1 个 AXI Interconnect IP，用于协议转换(AXI3<->AXI4)。
  • PL作为主机PS作为从机，PL向DDR中写数据，PS在DDR中读数据，实现PL到PS的数据传输，写入DDR后中断通知PS
  • AXI读通道和写通道是独立的支持同时读写
  • AXI4-Stream只有一个单一数据通道，和AXI4的写数据通道比较类似，突发长度无限制
  • AXI Interconnect IP和AXI SmartConnect IP都用于连接单个/多个存储器映射主从机。优先自动连接,资源不够只保留一种。AXI SmartConnect 能够提供更低的延迟。
  • AXI4-FULL支持突发传输(Burst Transfer)和非连续传输（支持间断传输）
  • AXI4(支持突发256)和AXI4-Lite(1个数据)都属于存储器映射
  • 写选通信号WSTRB：每8位数据对应一个写选通信号，指示数据中的哪些字节是有效的。
  • 突发传输：连续传输多个数据，突发长度最多256
  • 突发大小：突发中每次传输的数据字节数，用AxSIZE标识，突发大小为2^AxSIZE
  • 突发长度：突发传输中数据传输的次数（用AxLEN标识，标识值从0开始，实际突发长度为AxLEN+1）

地址对齐

每个数据传输单元的大小会与传输单元的边界对齐。例如，32位宽的数据传输通常以4字节为单位进行对齐。但在某些应用场景中，可能需要从非对齐地址开始数据传输突发。
对齐 指的是突发传输中起始地址与数据总线宽度之间的关系。
图中每行对应一次传输操作，根据地址与控制信息判断，阴影区域表示未传输的字节。
当起始地址 % 数据总线宽度（字节数） == 0时地址是对齐的

当地址非对齐时，第一次传输中，数据会被移位到对应的字节通道上。
由从机（Slave）或互连逻辑负责处理未对齐地址，通过 WSTRB 或 读取后的移位 来实现。
例如下面这个示例中，第一次传输的WSTRB = 4'b1110，屏蔽0x00地址上的数据，其余传输的WSTRB = 4'b1111。

第一次传输的WSTRB = 4'b1000，屏蔽0x04、0x05、0x06地址上的数据，其余传输的WSTRB = 4'b1111。

突发类型

AXI 定义三种突发类型，决定地址如何从当前 beat 计算到下一个 beat。

1. FIXED（固定地址）

  • 地址不变化，每个 beat 访问相同地址。
  • 典型用途：重复访问同一寄存器（如 FIFO 口）。
  • 例：起始地址 0x100，Burst Length=4，则地址序列：0x100, 0x100, 0x100, 0x100。

2. INCR（递增地址）

  • 地址按 beat 递增，递增量为 传输数据宽度（字节数）。
  • 这是最常用的类型（用于线性缓冲区、内存拷贝）。
  • 例：32位总线，起始地址 0x100，Burst Length=4：
    地址序列：0x100, 0x104, 0x108, 0x10C。
  • 特殊规则：INCR 突发可以跨越 1KB 边界，但协议限制不能跨越 4KB 地址边界（这是 AXI 的地址空间保护边界）。若突发会跨 4KB，则必须拆分。

3. WRAP（回环突发）

  • 主要用于缓存行填充（cache line fill）。
  • 地址递增，但当到达回环边界时，地址跳回起始地址对齐的边界。
  • 回环边界 = 突发总大小（Burst Size × Burst Length），并且必须是 2 的幂。

AXI的工作方式

AXI4和AXI-Lite包含5个独立的通道

读事务

  • 读地址通道
  • 读数据通道

写事务

  • 写地址通道
  • 写数据通道
  • 写响应通道
  • AXI4:由于读写地址通道是分离的，所以支持双向同时传输；突发长度最大256
  • AXI4-Lite:和AXI4比较类似，但是不支持突发传输
  • AXI4-Stream:只有一个单一数据通道，没有地址通道，和AXI4的写数据通道比较类似，突发长度不受限制。

AXI的通道定义

  • 每一个独立的通道都包含一组信息信号、VALID信号和READY信号用于提供双向的握手机制。
  • 信息源端使用VALID信号表示当前通道地址、数据和控制信息什么时候有效，
  • 目的端使用READY信号表示什么时候可以接收信息，
  • 读数据通道和写数据通道都包含一个LAST信号，用于表示传输的最后一个数据。
  • 读事务和写事务都有独立的地址通道，地址通道携带了请求所需的地址和控制信息
  • 读数据通道由从机发送给主机，包含了读数据和读响应的信息，读响应的信号用于表示读传输是否操作完成;
  • 写数据通道由主机发送给从机，包含了写数据，然后通过WSTRB信号表示当前数据的哪个字节有效
  • 写响应通道由从机发给主机，包含了写响应信号，用于表示当前写操作是否完成。

握手机制

  • 所有的五个通道都是通过相同的VALID/READY握手处理来传输地址、数据和控制信息
  • 双向握手的机制意味着主机和从机之间传输数据时，都可以控制传输的速率
  • 只有当VALID和READY同时为高电平时，传输才会发生。
    AXI4-Lite
  • 适用于当不需要AXI4完整功能的时候，一些简单的控制寄存器的读写
