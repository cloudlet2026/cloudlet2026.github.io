AMBA APB协议详解

第一章 概述

APB（Advanced Peripheral Bus，高级外围总线）是 ARM 公司 AMBA 总线体系结构中的一种低功耗、低复杂度的外设总线协议，主要用于连接慢速、低带宽的外设模块（如 UART、GPIO、I²C、定时器等）。

APB 协议有三个主要版本：

  • APB2（AMBA 2，1999年）：基础版本，传输固定为 2 个周期，从设备无等待能力，无错误报告
  • APB3（AMBA 3，2003年）：增强版本，增加 PREADY 支持从设备插入等待，增加 PSLVERR 支持错误报告
  • APB4（AMBA 4/5，2010年）：在 APB3 基础上，新增 PPROT 支持事务保护，新增 PSTRB 支持写选通（字节粒度写入）

第二章 APB2 协议详解

APB2 是 APB 协议的早期版本，特点是结构简单、传输固定为 2 个 PCLK 周期，适用于不需要等待和错误报告机制的简单外设。

2.1 APB2 信号列表

APB2 共有 8 个信号（不含时钟和复位），比 APB3 少了 PREADY 和 PSLVERR。

信号名 方向 宽度 描述
PCLK - 1 时钟（所有信号上升沿采样）
PRESETn - 1 低电平有效复位
PADDR Master→Slave 32 地址总线（固定 32 位）
PWDATA Master→Slave 32 写数据总线（固定 32 位）
PWRITE Master→Slave 1 1=写，0=读
PSEL Master→Slave 1 片选（每个 Slave 独立）
PENABLE Master→Slave 1 传输使能（ACCESS 阶段拉高）
PRDATA Slave→Master 32 读数据总线（固定 32 位）

核心特征：APB2 无 PREADY（从设备不能插入等待），无 PSLVERR（无错误报告），所有传输固定为 2 个 PCLK 周期。

2.2 APB2 状态机

APB2 使用两状态状态机，ACCESS 阶段固定持续 1 个周期，无法延长。

状态 PSEL PENABLE 持续时间 说明
IDLE 0 0 空闲 默认状态，无传输进行
SETUP 1 0 1 个周期 地址和控制信号建立阶段
ACCESS 1 1 1 个周期 数据传输阶段，固定一拍完成

关键约束：APB2 中从设备必须在 ACCESS 周期内完成读写响应，不能延长总线周期。如果从设备响应较慢，需要内部插入等待状态，但总线层面无法扩展。

2.3 APB2 读传输时序

时序解读（分步说明）：

时刻 事件 详细说明
T0→T1 传输发起 主机在 T1 上升沿将 PSEL 拉高，PENABLE 保持为 0，PADDR 上驱动有效地址，PWRITE=0 表示读操作
T1→T2 SETUP 阶段 地址和控制信号保持稳定，从设备采样地址并进行地址译码
T2↑ 进入 ACCESS 主机将 PENABLE 拉高，从设备在 T2 上升沿捕获到有效传输（PSEL=1, PENABLE=1）
T2→T3 ACCESS 阶段 从设备必须在当前 T2 周期内将读数据驱动到 PRDATA 上（组合逻辑输出）
T3↑ 传输结束 主机在 T3 上升沿采样 PRDATA，同时将 PSEL 和 PENABLE 拉低，回到 IDLE

设计要点：PRDATA 在 APB2 中通常使用组合逻辑输出，确保在 ACCESS 周期内数据立即有效。寄存器输出会导致一个周期的额外延迟，造成主机采样到错误数据。

2.4 APB2 写传输时序

时序解读（分步说明）：

时刻 事件 详细说明
T0→T1 传输发起 主机在 T1 上升沿将 PSEL 拉高，PENABLE=0，PADDR 驱动地址，PWRITE=1 表示写操作
T1→T2 SETUP 阶段 PWDATA 上同时驱动有效写数据。注意：写数据在 SETUP 阶段就已有效，为从设备提供更充分的建立时间
T2↑ 进入 ACCESS 主机将 PENABLE 拉高，从设备在 T2 上升沿捕获地址、控制信号和 PWDATA
T2→T3 ACCESS 阶段 从设备必须在当前 T2 周期内将 PWDATA 写入内部寄存器
T3↑ 传输结束 主机将 PSEL 和 PENABLE 拉低，从设备应在 T3 上升沿之前完成写入

设计要点：APB2 写操作中，PWDATA 在 SETUP 阶段就已经有效，从设备可以在 T1→T2 期间就开始采样 PWDATA，不必等到 ACCESS 阶段，有利于高速设计。

第三章 APB3 协议详解

APB3 是 APB2 的增强版本，在保持低功耗、低复杂度的基础上，增加了从设备等待能力（PREADY） 和错误报告机制（PSLVERR），适用于需要可变响应时间的复杂外设。

3.1 APB3 信号列表

APB3 共有 10 个信号（不含时钟和复位），比 APB2 多了 PREADY 和 PSLVERR。

信号名 方向 宽度 描述
PCLK - 1 时钟（所有信号上升沿采样）
PRESETn - 1 低电平有效复位
PADDR Master→Slave 32/64 地址总线（可选宽度）
PWDATA Master→Slave 32/64 写数据总线（可选宽度）
PWRITE Master→Slave 1 1=写，0=读
PSEL Master→Slave 1 片选（每个 Slave 独立）
PENABLE Master→Slave 1 传输使能（ACCESS 阶段拉高）
PRDATA Slave→Master 32/64 读数据总线（可选宽度）
PREADY Slave→Master 1 0=插入等待，1=就绪（新增）
PSLVERR Slave→Master 1 1=传输出错（新增）

核心增强：

  • PREADY：从设备可通过拉低 PREADY 来延长 ACCESS 阶段，插入任意数量的等待周期
  • PSLVERR：从设备可在传输完成时返回错误状态，仅当 PREADY=1 的周期有效

3.2 APB3 状态机

APB3 使用三状态状态机，ACCESS 阶段可通过 PREADY=0 无限期延长。

状态 PSEL PENABLE 持续时间 说明
IDLE 0 0 空闲 默认状态，无传输进行
SETUP 1 0 1 个周期 地址和控制信号建立阶段，固定 1 拍
ACCESS 1 1 1 个或多个周期 数据传输阶段，PREADY=0 时自旋等待，PREADY=1 时完成并回到 IDLE

关键约束：

  • SETUP 阶段必须恰好 1 个周期，不能延长
  • 只有 ACCESS 阶段可以通过 PREADY 延长
  • PREADY 在 ACCESS 阶段被采样，在 SETUP 阶段可以被忽略

3.3 APB3 无等待读传输（PREADY=1）

这是 APB3 中最简单、最高效的传输模式，与 APB2 时序完全相同，但增加了 PREADY 信号。

时序解读（分步说明）：

时刻 事件 详细说明
T1↑ SETUP 开始 PSEL 拉高，PENABLE=0，PADDR 驱动地址，PWRITE=0（读）
T1→T2 SETUP 阶段 地址和控制信号建立，从设备进行地址译码
T2↑ ACCESS 开始 PENABLE 拉高，从设备采样到有效传输
T2→T3 ACCESS 阶段 PREADY=1 表示从设备已就绪，在 T2 周期内将数据驱动到 PRDATA
T3↑ 传输结束 主机采样 PRDATA，同时拉低 PSEL 和 PENABLE

设计要点：无等待读传输总长度为 2 个周期，与 APB2 完全一致。PREADY 在 T2 周期内保持为 1。

3.4 APB3 带等待读传输（PREADY=0 插入等待）

当从设备需要更多时间准备数据时，可通过拉低 PREADY 来延长 ACCESS 阶段。

时序解读（分步说明）：

时刻 事件 详细说明
T1↑ SETUP 开始 主机发起读传输，PSEL 拉高，地址驱动到 PADDR
T1→T2 SETUP 阶段 从设备采样地址，判断需要额外时间，准备插入等待
T2↑ ACCESS 开始 PENABLE 拉高，进入 ACCESS 阶段
T2→T3 第 1 拍等待 从设备将 PREADY 驱动为 0，主机检测到后保持在 ACCESS 状态，不采样 PRDATA
T3→T4 第 2 拍等待 PREADY 仍然为 0（本例等待 2 拍），主机继续自旋，PRDATA 无效
T4→T5 数据就绪 从设备将 PREADY 拉高，同时将有效数据驱动到 PRDATA
T5↑ 传输结束 主机采样 PRDATA，拉低 PSEL 和 PENABLE，传输完成

设计要点：

  • 等待拍数由从设备通过 PREADY 控制，可任意长度
  • PRDATA 仅当 PREADY=1 的周期才有效
  • 主机在等待期间保持所有输出信号（PSEL、PENABLE、PADDR、PWDATA）稳定
  • 本例总周期数 = 1（SETUP）+ 1（最后 ACCESS）+ 2（等待）= 4 个周期

3.5 APB3 无等待写传输（PREADY=1）

时序解读（分步说明）：

时刻 事件 详细说明
T1↑ SETUP 开始 PSEL 拉高，PENABLE=0，地址驱动，PWRITE=1（写），PWDATA 同时有效
T1→T2 SETUP 阶段 地址、控制信号和写数据建立，从设备采样
T2↑ ACCESS 开始 PENABLE 拉高，从设备捕获到有效传输
T2→T3 ACCESS 阶段 PREADY=1，从设备在 T2 周期内将 PWDATA 写入内部寄存器
T3↑ 传输结束 主机拉低 PSEL 和 PENABLE

设计要点：与 APB2 写传输完全一致，PWDATA 在 SETUP 阶段就已有效。总长度为 2 个周期。

3.6 APB3 带等待写传输（PREADY=0 插入等待）

写操作同样支持 PREADY 等待，适用于慢速存储设备或需要较长写入时间的寄存器。

时序解读（分步说明）：

时刻 事件 详细说明（修正版）
T1↑ SETUP 开始 主机发起写传输，PSEL 拉高，PENABLE=0，PADDR 驱动地址，PWRITE=1。PWDATA 同时驱动有效写数据
T1→T2 SETUP 阶段 地址、控制信号和写数据均保持稳定，从设备采样地址并判断需要插入等待
T2↑ ACCESS 开始 PENABLE 拉高，从设备在 T2 上升沿捕获到有效传输（PSEL=1, PENABLE=1, PWRITE=1）
T2→T3 第 1 拍等待 从设备将 PREADY 驱动为 0，主机保持在 ACCESS 状态，PWDATA 保持不变
T3→T4 第 2 拍等待 PREADY 仍然为 0（本例等待 2 拍），主机继续自旋，PWDATA 仍然保持不变
T4→T5 写完成 从设备在 T4 周期内将 PREADY 拉高为 1，并在同一周期内将 PWDATA 上的数据写入内部寄存器
T5↑ 传输结束 主机在 T5 上升沿采样到 PREADY=1，确认写入完成，同时将 PSEL 和 PENABLE 拉低，回到 IDLE 状态

设计要点：

  • 写数据在 SETUP 阶段就已驱动，等待期间 PWDATA 保持不变
  • 只有 PREADY=1 的周期才执行实际写入
  • 若 PREADY=0 时写入，会导致数据被重复写入或时序错误

3.7 APB3 错误传输（PSLVERR）

APB3 新增 PSLVERR 信号，允许从设备在传输出错时向主机报告。典型错误场景包括：

  • 访问非法地址（地址译码失败）
  • 对只读寄存器执行写操作
  • 对只写寄存器执行读操作
  • 从设备内部异常（如 FIFO 溢出）

PSLVERR 使用规则：

规则 说明
有效窗口 仅在 ACCESS 阶段最后一个周期有效（即 PENABLE=1 且 PREADY=1 的周期）
读操作 PRDATA 上的数据可能无效，主机应忽略
写操作 从设备不应更新内部寄存器
等待场景 若 PREADY=0 插入等待，PSLVERR 必须在 PREADY=1 的最后一个周期才有效

带等待1拍的错误写传输时序：

时序解读：

时刻 事件 详细说明
T1↑ SETUP 开始 主机发起传输写，PSEL 拉高，PENABLE=0，地址/控制信号驱动到总线上
T1→T2 SETUP 阶段 地址和控制信号保持稳定，从设备采样地址并开始解码，发现非法或错误条件，决定插入等待
T2↑ ACCESS 开始 PENABLE 拉高，进入 ACCESS 阶段。从设备将 PREADY 驱动为 0，表示尚未准备好完成传输
T2→T3 第 1 拍等待 PREADY=0，主机检测到后保持在 ACCESS 状态自旋，不结束传输。从设备利用此周期准备错误响应（PSLVERR 在此期间保持为 0，无效）
T3↑ 等待结束 / 传输完成 从设备将 PREADY 拉高为 1，同时将 PSLVERR 驱动为 1（因为这是传输的最后一个周期）
T3→T4 ACCESS 阶段（最后有效周期） PREADY=1，PSLVERR=1，从设备在 T3 周期内将错误状态保持稳定
T4↑ 传输结束 主机在 T4 上升沿采样到 PREADY=1 且 PSLVERR=1，识别出本次传输失败，随后将 PSEL 和 PENABLE 同时拉低，回到 IDLE 状态

五、APB4 协议详解

APB4 是 APB3 的增强版本，APB4 在保持 APB3 全部特性（PREADY 等待、PSLVERR 错误报告）的基础上，新增了两个核心信号，主要用于系统安全保护和精细化数据写入。

5.1 APB4 新增信号

APB4 在 APB3 的 10 个信号基础上，新增了 2 个信号，共计 12 个信号：

信号名 方向 宽度 描述
PPROT Master→Slave 3 事务保护信号（新增）
PSTRB Master→Slave 数据宽度/8 写选通信号（新增）

注意：APB4 完全向后兼容 APB3，APB3 Slave 可以直接接入 APB4 Master，只需将 PPROT 和 PSTRB 信号忽略或不连接即可。

5.2 PPROT：事务保护信号

PPROT[2:0] 是一个 3 位保护信号，由主机在 SETUP 阶段驱动，用于向从设备告知本次访问的权限和类型，使从设备能够根据访问属性决定是否允许本次操作，是实现系统安全隔离的基础。

5.2.1 PPROT 位定义

位 名称 含义 说明
PPROT[0] Privileged 0 = 普通访问 (Normal) 对应非特权模式（如用户态）的访问请求
1 = 特权访问 (Privileged) 对应特权模式（如操作系统内核）的访问请求
PPROT[1] Secure 0 = 安全访问 (Secure) 属于安全世界（TrustZone 安全域）的访问
1 = 非安全访问 (Non-secure) 属于非安全世界（普通应用域）的访问
PPROT[2] Instruction/Data 0 = 数据访问 (Data) 访问数据区域（如全局变量、堆栈）
1 = 指令访问 (Instruction) 访问指令区域（如代码段取指）

5.2.2 典型应用场景

场景 PPROT 值 说明
用户态程序读取 UART 数据 3’b000 普通 + 非安全 + 数据访问
内核驱动配置 DMA 控制器 3’b001 特权 + 非安全 + 数据访问
安全 OS 访问安全密钥寄存器 3’b010 特权 + 安全 + 数据访问
CPU 从外设 ROM 取指令 3’b101 特权 + 非安全 + 指令访问

5.3 PSTRB：写选通信号

PSTRB[n:0] 是一个写选通信号，由主机在写传输中驱动，用于指示 PWDATA 上的哪些字节是有效的。

  • 宽度：等于数据总线宽度除以 8（例如 32 位数据总线 → PSTRB 为 4 位）
  • 对应关系：PSTRB[n] 对应 PWDATA[(8n+7) : (8n)]，即第 n 个字节
  • 有效电平：高电平有效（1 = 该字节有效，应当写入；0 = 该字节无效，不写入）

5.3.1 PSTRB 与 PWDATA 的对应关系

数据总线宽度 PSTRB 宽度 对应关系
32 位 4 位 PSTRB[0]→PWDATA[7:0]，PSTRB[1]→PWDATA[15:8]，PSTRB[2]→PWDATA[23:16]，PSTRB[3]→PWDATA[31:24]
64 位 8 位 PSTRB[0]→PWDATA[7:0]，…，PSTRB[7]→PWDATA[63:56]

5.3.2 PSTRB 的典型用法

场景 PSTRB 值（32位总线） 效果
写 32 位完整数据 4’b1111 全部 4 个字节均写入
写最低字节（Byte 0） 4’b0001 只更新 PWDATA[7:0]，其他字节不变
写高 16 位（Byte 2-3） 4’b1100 只更新 PWDATA[31:16]，其他字节不变
写中间字节（Byte 1） 4’b0010 只更新 PWDATA[15:8]，其他字节不变

5.4 APB4 时序波形

① APB4 无等待写传输（带 PSTRB 和 PPROT）

时序解读：

时刻 事件 详细说明
T1↑ SETUP 开始 与 APB3 相同，PSEL 拉高，PENABLE=0，PADDR 和 PWRITE 被驱动。PPROT 和 PSTRB 也在此时被驱动，与地址和控制信号一同建立
T1→T2 SETUP 阶段 地址、控制、保护属性、写选通和写数据均保持稳定，从设备采样所有信息
T2↑ ACCESS 开始 PENABLE 拉高，进入 ACCESS 阶段
T2→T3 ACCESS 阶段 PREADY=1，从设备在 T2 周期内根据 PPROT 检查权限，根据 PSTRB 选择性写入 PWDATA 的有效字节
T3↑ 传输结束 主机拉低 PSEL 和 PENABLE

关键要点：PPROT 和 PSTRB 与 PADDR 和 PWRITE 同时驱动，在 SETUP 阶段就已建立，与 APB3 的地址/控制信号时序完全一致。

② APB4 带等待写传输（PREADY=0 插入 1 拍）

时序解读：

时刻 事件 详细说明
T1↑ SETUP 开始 与无等待相同，PPROT 和 PSTRB 在 SETUP 阶段驱动
T1→T2 SETUP 阶段 所有信号建立
T2↑ ACCESS 开始 PENABLE 拉高，从设备采样，决定插入等待
T2→T3 第 1 拍等待 PREADY=0，PPROT 和 PSTRB 必须保持不变，如同 PADDR 和 PWDATA
T3→T4 写完成 PREADY=1，从设备根据 PPROT 检查权限，根据 PSTRB 执行选择性写入
T4↑ 传输结束 主机拉低 PSEL 和 PENABLE

关键要点：在等待期间，所有输出信号（包括 PPROT 和 PSTRB）都必须保持稳定，直到传输完成。

6.2 常见问题 Q&A

Q1：APB3 Slave 中，PREADY 在 SETUP 阶段应该是什么值？

A：PREADY 只在 ACCESS 阶段被主机采样，在 SETUP 阶段可以任意值（通常建议保持为 1）。但为规范起见，建议 PREADY 仅在 ACCESS 阶段根据等待逻辑驱动，SETUP 阶段输出默认值 1。

Q2：PSLVERR 在等待期间（PREADY=0）可以拉高吗？

A：不可以。PSLVERR 仅在传输的最后一个周期有效，即 PENABLE=1 且 PREADY=1 的周期。在等待期间（PREADY=0），PSLVERR 必须为 0。

Q3：APB3 Master 在等待期间，地址和数据信号需要保持不变吗？

A：必须保持不变。在 PREADY=0 的等待周期中，Master 必须保持 PSEL、PENABLE、PADDR、PWRITE、PWDATA 稳定，不能改变，直到 PREADY 拉高且传输完成。

Q4：APB2 Slave 能否直接接到 APB3 Master 上？

A：可以。只需将 APB2 Slave 的 PREADY 输入固定接 1，PSLVERR 固定接 0 即可（或内部硬连接）。传输时序完全兼容。

Q5：APB3 中，读数据（PRDATA）应该用组合逻辑还是时序逻辑输出？

A：推荐组合逻辑输出。因为 PRDATA 需要在 ACCESS 阶段（PREADY=1 的周期）内被主机在下一个上升沿采样。若使用寄存器输出，会导致一个周期的延迟，主机可能采样到旧数据。若确实需要使用寄存器输出，则需要额外插入等待周期。

Q6：APB4 的 PPROT 和 PSTRB 是在哪个阶段驱动的？

A：PPROT 和 PSTRB 与 PADDR、PWRITE 同时在 SETUP 阶段驱动，并保持到整个传输结束。在等待期间（PREADY=0），它们也必须保持稳定不变。

Q7：APB3 Slave 可以直接接到 APB4 Master 上吗？

A：可以。APB4 完全向后兼容 APB3。APB3 Slave 会忽略 PPROT 和 PSTRB 信号，APB4 Master 会将这些信号驱动为默认值（PPROT=3’b000，PSTRB=全 1），传输时序完全一致。

Q8：PSTRB 在写传输中必须使用吗？

A：不必须。PSTRB 是可选的。对于简单外设，可以固定为全 1（所有字节均有效），表示完整的 32/64 位写入。只有在需要字节粒度写入控制时才需要使用 PSTRB。
