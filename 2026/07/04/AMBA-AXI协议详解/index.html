AMBA AXI协议详解

发表于2026-07-04|更新于2026-07-25
|浏览量:

本文由AI生成，不具备参考价值。

AXI（Advanced eXtensible Interface，高级可扩展接口） 是 ARM 公司 AMBA 总线体系结构中的一种高性能、高带宽、低延迟的系统总线协议，主要用于连接高速、高带宽的模块（如 CPU、DDR 控制器、GPU、DMA、高速网络接口等）。

一、AXI4 核心信号（5通道独立）

(主机发出地址)		 Master Slave 主机告诉从机：我要读哪块地址。
读数据 R Slave → Master
(从机返回数据)		 Slave Master 从机把读到的数据和响应送回主机。

(主机发出地址)		 Master Slave 主机告诉从机：我要往哪块地址写。
写数据 W Master → Slave
(主机发出数据)		 Master Slave 主机把要写入的实际数据发给从机。

二、AXI4 架构总览

五个独立通道

AXI4最核心的设计思想是将地址/控制信号与数据信号分离到不同的通道，每个通道独立握手，互不阻塞。

三、AXI4 握手时序详解

三种握手方式

① VALID 先变高 READY 后变高

② READY 先变高 VALID 后变高

③ VALID 和 READY 信号同时变高

  • 协议的唯一铁律：VALID信号不可以依赖READY信号。也就是说，发送方（Source）只要把数据/地址准备好了，就必须把VALID拉高，不能等看到对方的READY为高再拉VALID（否则会引发死锁风险）。

当前信号是谁发出来的，就由谁来提供数据（DATA）/状态（VALID）。

发送方角色 典型通道（读/写） 数据/地址来源 工程上常用的策略 原因
Master（主机） AW（写地址）、W（写数据）、AR（读地址） CPU/DMA内部逻辑，响应极快，一个周期内就能算出来。 倾向于"同时拉高"或"READY先高" 主机通常频率高、逻辑简单，为了追求极致带宽，希望一拍之内完成握手，绝不多等。
Slave（从机） R（读数据）、B（写响应） 外部存储器（DDR/Flash）、慢速外设、内部RAM（可能需多个周期读出）。 几乎全是"VALID先拉高" 从机不知道何时能取到数据。当它终于取到数据放在总线上时，必须第一时间把VALID置1，然后等着主机（Master）腾出手来把READY拉高把数据拿走。

四、AXI4 读事务时序

时序解读：

  • T0→T3：Master拉高ARVALID，Slave在T2周期拉高ARREADY，地址握手在T3上升沿完成。
  • T3→T6：Slave准备数据，在T4周期拉高RVALID，Master在T6周期拉高RREADY，数据握手在T7上升沿完成（首次数据）。
  • 地址通道和数据通道完全独立：地址握手在T3完成，数据握手在T7完成，中间有4拍延迟（这是Slave的读延迟）。
  • 突发传输：RLAST在最后一拍数据时拉高（图中A3是最后一拍）。

五、AXI4 写事务时序

时序解读：

  1. AW和W完全独立：Master在T1发地址，在T3才发数据。Slave在T3接收地址，在T5接收数据。
  2. WLAST：在最后一拍数据时拉高。
  3. B通道：Slave在收到所有数据并写入完成后才拉高BVALID返回响应。BVALID在T13或更晚出现。

六、AXI4 高级特性

超前传输（Outstanding）

Outstanding 是指Master在等待第一个事务的响应（R或B）之前，就可以继续发送后续事务的地址。

时序解读：

  • Master在T1、T4发送2个地址，不需要等待读数据返回。
  • Slave可以在后续连续返回数据（顺序或乱序，由ID控制）。
  • Outstanding深度 = 2（本例中）。

乱序传输（Out of Order）与交织传输（Interleaving）

  • 乱序传输（Out-of-Order）：

  • 作用对象：不同ID（或者不同通道）的整个突发传输（Burst）。

  • 定义：主机（Manager）发出的多个写/读请求（地址通道AR/AW）的顺序，与数据通道（W/R）上完成传输的顺序，或者响应通道（B）上返回的顺序不一致。

  • 本质：请求顺序的重排。目的是让先发起但被阻塞的慢操作，让位给后发起但准备就绪的快操作。

  • 交织传输（Interleaving）：

  • 作用对象：同一个ID的单个突发传输内部的各个数据节拍（Beats）。

  • 定义：在读取回传数据（R通道）时，来自不同ID的多个突发传输的数据节拍，可以交错地在总线上进行传输。

  • 本质：数据包的分割与穿插。目的是防止某个长突发长时间占据总线，导致其他ID的短突发饿死。

比较维度 乱序传输 （Out-of-Order） 交织传输 （Interleaving）
粒度 Burst 级别（整个突发完成顺序变化） Beat 级别（突发内部单个数据传输顺序变化）
依赖标识 ARID / AWID（事务ID） RID（读数据ID，必须与ARID匹配）
响应通道（B） 允许乱序（BID 顺序与 AWID 不同） 不涉及（写响应没有交织概念，B通道顺序与写数据交织无关）
关键前提 从机必须声明支持 OutOfOrder 能力 从机必须声明支持 Interleaving 能力
限制规则 相同 ID 的突发必须严格保序 相同 ID 的 Beat 必须连续传输（中间不能插入其他ID的Beat）

不同ID的事务可以乱序完成，但同一ID的事务必须严格保序完成。
这条铁律同时适用于读（R通道）和写（B通道），是AXI乱序机制的最高准则。
通常支持交织就自然支持乱序，复杂的高性能从设备常常会同时具备这两个特性。

写数据乱序（AXI4 vs AXI3）

这是AXI4与AXI3的重大区别，也是乱序传输中最容易被误解的地方。AXI4中写通道的乱序行为需要分两个层面来看：

通道 是否允许乱序 原因
写数据（W通道） 不允许 AXI4移除了WID信号，从机只能按物理接收顺序匹配数据与地址
写响应（B通道） 允许 响应自带BID标签，从机可独立返回任何ID的完成状态

容易混淆点： “写通道乱序"≠"写数据乱序”。写数据必须顺序发送，但写响应可以乱序返回。

乱序 + 交织场景示例

场景一：仅支持乱序，不支持交织（普通高性能模式）

场景二：乱序 + 交织 同时支持（极致性能模式）

在实际 SOC 设计中，交织虽然能提高吞吐，但会极大增加从机内部的状态机复杂度（需要维护多个 RID 的当前计数值）。因此，AXI4 协议规定，交织深度（Interleaving Depth）必须由从机在 RUSER 或配置参数中声明，且主机在发起传输时，需要确保接收缓冲区能够容纳交织带来的乱序数据重组压力。

七、AXI4的Outstanding深度控制

Outstanding是AXI实现高带宽的核心特性之一。它打破了传统总线"发起请求→等待响应→再发起请求"的串行模式，允许Master在未收到前一个事务响应时，就连续发出多个新事务的地址。这样，即使访问DDR这类高延迟外设，总线也能在等待期间处理其他请求，从而隐藏访问延迟，大幅提升带宽利用率。

要正确实现Outstanding，关键在于深度控制，这涉及三个层面的协同配合：

Master的发送能力（MAX_OUTSTANDING）

Master内部维护一个计数器，记录当前"在途"（已发地址但未收响应）的事务数量。这个计数器会随着地址握手成功而增加，随着响应完成而减少。

MAX_OUTSTANDING的上限通常取决于Master内部为保存在途事务信息而配备的缓冲区大小，每个在途事务需要缓存地址、控制信息以及后续返回的数据。

Interconnect的转发能力（ISSUE_CAPACITY）

总线互联结构（Interconnect）本身也有处理能力上限，需要能够容纳和转发来自多个Master的Outstanding请求。如果多个Master同时发送大量Outstanding请求，Interconnect内部的缓冲队列可能溢出。

Slave的接收能力（ACCEPT_CAPACITY）

Slave通常有自己的队列来缓存接收到的请求。如果Slave的Outstanding能力为N，意味着它可以同时处理N个未完成的事务。如果Master发送速度过快，超过了Slave的处理能力，Slave可以通过拉低对应的AxREADY信号来进行反压，告诉Master"我暂时处理不了新的请求了"。

八、AXI4-Lite Slave RTL代码

AXI4-Lite是AXI的简化版，不支持突发传输（LEN固定为0），但保留了完整的5通道握手。它最适合作为寄存器的配置接口。

AXI4-Lite Slave 代码要点：

  1. 地址和数据通道独立握手：状态机同时监测AWVALID和WVALID，支持地址先到或数据先到。
  2. 写响应延迟：必须等到地址和数据都握手完成后，才进入WR_RESP状态返回BVALID。
  3. WSTRB字节选通：支持部分字节写入，适用于对32位寄存器中个别字节的更新。
  4. 读数据零延迟：使用组合逻辑，ARVALID有效时RDATA在同一周期输出（适用于寄存器读）。

十、AXI4-Stream 协议详解

AXI4-Stream 与前面讲的 AXI4 最大的区别是：没有地址通道。它不涉及"读某个地址"或"写某个地址"，只有纯粹的数据流发送与接收。这让它非常适合视频流、音频流、高速ADC/DAC数据采集等场景。

核心信号

信号 方向 描述
ACLK - 全局时钟，上升沿采样
ARESETn - 低电平复位，复位期间TVALID必须为0
TVALID Master→Slave 主机数据有效标志
TREADY Slave→Master 从机准备好接收
TDATA Master→Slave 数据总线（宽度可配置）
TSTRB Master→Slave 字节类型标志（1=数据字节，0=位置字节）
TKEEP Master→Slave 字节有效标志（1=该字节需传输到目的地）
TLAST Master→Slave 当前Packet的最后一个数据
TID Master→Slave 数据流标识符（可选）
TDEST Master→Slave 路由信息（可选）
TUSER Master→Slave 用户自定义信息（可选）

握手机制

AXI4-Stream 的握手与 AXI4 一致，遵循 VALID/READY 双向握手机制：

关键规则：

  1. Master不能等TREADY拉高才拉TVALID。一旦TVALID拉高，必须保持直到握手完成（TVALID && TREADY）。
  2. Slave可以等TVALID拉高再拉TREADY，也可以提前拉TREADY。
  3. TLAST 在Packet最后一个数据时拉高，表示本次流传输结束。
