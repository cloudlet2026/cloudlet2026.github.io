1.1 运行环境差异

特性 桌面C/C++ IC验证嵌入式C/C++
入口函数 main(int argc, char *argv[]) void main(void) 或 _start（由链接脚本指定）
堆栈初始化 由OS完成 必须由启动代码（crt0）手动设置SP（栈指针）

异常处理 try/catch/信号 通常禁用C++异常，使用返回值或断言

1.2 编译工具链差异

  • 使用 交叉编译工具链（如 riscv64-unknown-elf-gcc, arm-none-eabi-gcc）。
  • 关键选项：
    • -march / -mcpu：指定CPU架构。
    • -ffreestanding：表示无宿主OS，不依赖标准库启动代码。
    • -nostdlib：不自动链接标准库启动文件（需自定义链接脚本）。

第2章 链接脚本（Linker Script）—— 芯片的内存地图

链接脚本是验证C代码能否正确运行于CPU上的地基。它决定了代码、数据、堆栈在物理/虚拟地址空间中的摆放位置。

2.1 必须理解的内存区域

  • ITCM / IRAM：指令紧耦合内存，放.text段（代码）。
  • DTCM / DRAM：数据紧耦合内存，放.data（初始化为非零的全局变量）、.bss（零初始化全局/静态变量）。
  • ROM / Flash：只读区，放常量.rodata。
  • Peripheral Region：外设寄存器MMIO地址（如UART、GPIO、中断控制器）。

3.1 关键初始化步骤

  1. 关中断（防止未初始化时触发异常）。
  2. 设置栈指针（SP）：将_stack_top值加载到sp寄存器。

第7章 C++在IC验证中的有限但强大应用

虽然验证中的固件主体多用C，但C++的封装和RAII（资源获取即初始化）特性对提高验证代码的可维护性非常有价值。但需要严格控制C++特性的使用。

7.1 允许使用的C++特性

  • 类（用于封装寄存器组）
  • 命名空间（避免冲突）
  • 内联函数（inline）
  • 模板（用于实现位域操作的编译期计算）
  • 重载（运算符或函数）

7.2 明确禁用的C++特性

  • 异常（try/catch/throw）：运行时开销大，且需要 __cxa_throw 等GCC运行时支持（难以在裸机部署）。
  • RTTI（运行时类型识别，typeid/dynamic_cast）：增加代码体积和内存开销。
  • 标准库容器（std::vector, std::string）：依赖堆分配，容易碎片化且实时性不可预测。
  • 全局对象构造（静态初始化）：构造顺序不确定，且会引入 __cxx_global_var_init，在启动代码中难以控制。
