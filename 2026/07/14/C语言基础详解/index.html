第1章：C语言概述与环境搭建

1.1 什么是C语言

C语言是一门结构化、编译型、高效的编程语言，诞生于1972年，由Dennis Ritchie在贝尔实验室开发。它兼具高级语言的易读性和汇编语言的底层控制能力，是操作系统、嵌入式系统、驱动程序、数据库等底层软件的首选语言。

特点：

  • 直接操作内存（指针）
  • 语法简洁，但灵活性强
  • 可移植性好（标准库支持）
  • 生成代码高效，接近硬件

第7章：指针深度剖析

指针是C语言的灵魂，它存储变量的内存地址，允许直接操作内存。

7.1 指针的基本概念

  • 地址：内存中每个字节都有一个编号（地址）
  • 指针变量：专门存储地址的变量
  • 取地址符 &：获取变量的地址
  • 解引用 *：通过地址访问该地址上的数据

指针的类型很重要： int * 只能指向 int，char * 只能指向 char，类型决定了指针运算的步长和解释方式。

7.2 指针的声明与初始化

NULL指针： 指向地址0，解引用NULL会导致程序崩溃（段错误），常用来检查指针是否有效。

7.3 指针运算

指针支持加减整数，移动步长由类型大小决定。

关键点： p+1 不是地址+1，而是地址 + sizeof(int) 字节。

7.4 指针与数组

数组名在大多数情况下退化为指向首元素的指针，但二者有区别：

  • sizeof(arr) 返回整个数组字节数
  • sizeof(p) 返回指针本身大小（8或4字节）

第8章：动态内存分配

静态数组大小固定，动态内存允许程序在运行时按需分配，由程序员手动管理。

8.1 堆内存 vs 栈内存

特性 栈（Stack） 堆（Heap）
分配方式 自动（函数调用） 手动（malloc等）
大小 较小（默认几MB） 较大（系统内存）
生命周期 函数结束自动释放 直到free释放
管理 编译器自动 程序员负责

8.2 malloc - 分配内存

malloc(size_t size) 分配 size 字节，返回 void*，失败返回 NULL。

注意： malloc 不会初始化内存，内容随机（垃圾值）。

8.3 calloc - 分配并清零

calloc(count, size) 分配 count 个大小为 size 的元素，并全部初始化为0。

8.4 realloc - 重新调整大小

扩大或缩小已分配的内存块，尽可能原地扩展，否则移动数据。

危险： 若 realloc 失败返回 NULL，原指针仍有效，所以不要直接 arr = realloc(arr, ...)（会丢失原指针）。

8.5 free - 释放内存

常见错误：

  • 重复 free 同一块内存 → 崩溃
  • 释放后继续使用（悬空指针）
  • 忘记释放 → 内存泄漏

malloc 后忘记检查返回值

避坑： 总是检查 malloc/calloc/realloc 的返回值。

realloc 失败时原指针丢失

避坑： 使用临时指针。

重复释放（Double Free）

避坑： 释放后立即置 NULL。

使用已释放的内存（悬空指针）

避坑： 释放后置 NULL，或在任何 free 后不再使用该指针。

方式1：指针数组（行不连续，但灵活）

方式2：连续内存（一行存放，缓存友好）

第10章：预处理指令

10.1 #include - 文件包含

插入头文件内容，有两种形式：

  • #include <stdio.h> ：从系统标准路径搜索
  • #include "myheader.h" ：先从当前目录搜索，再到系统路径

头文件的作用： 声明函数原型、宏定义、类型定义，供多个 .c 文件共享。

10.2 #define - 宏定义

简单的文本替换，无类型检查。

宏的注意事项：

  • 每个参数都要加括号
  • 整个宏体加括号
  • 多行宏用 \ 续行
  • 宏名通常大写以区分函数

取消宏定义：

宏参数未加括号导致运算顺序错误

避坑： 宏体和每个参数都加括号。

宏的副作用（重复求值）

避坑： 对于有副作用的参数，不要使用宏，改用内联函数。

宏中分号问题

避坑： 宏不要包含末尾分号，调用者自己加。

第11章：文件操作

常用模式：

11.3 行读写 - fgets / fputs

11.4 格式化读写 - fprintf / fscanf

像 printf/scanf 一样读写文件。

11.5 二进制读写 - fread / fwrite

直接读写内存块，效率高，适合存储结构体等复杂数据。

返回值： fread/fwrite 返回成功读写的元素个数，可用于判断是否读到末尾。

11.6 文件定位 - fseek / ftell / rewind

  • fseek(FILE *fp, long offset, int whence)：移动文件位置指针
    • whence：SEEK_SET（开头）、SEEK_CUR（当前位置）、SEEK_END（末尾）
  • ftell(FILE *fp)：返回当前偏移量
  • rewind(FILE *fp)：回到文件开头（等价于 fseek(fp, 0, SEEK_SET)）

11.7 文件操作常见错误

  • 忘记检查 fopen 返回值
  • 忘记 fclose（可能导致数据未写入）
  • 读写模式不匹配（如用"r"打开不存在的文件）
  • 二进制文件用文本模式打开（Windows下换行符转换问题）

文本模式和二进制模式混用（Windows）

避坑： 对于非文本数据（结构体、图片等），始终使用二进制模式 "rb"/"wb"。

fgets 保留换行符

避坑： 使用 strcspn 去除换行。

21.3 堆内存管理底层原理

malloc 背后依赖 brk/sbrk（小内存）或 mmap（大内存）。

内存分配器策略（ptmalloc2）：

  • 小对象（< 128KB）使用 brk，复用空闲块
  • 大对象使用 mmap，释放时立即归还操作系统

第22章：内存对齐与缓存优化

22.1 结构体内存对齐

CPU访问未对齐的内存可能性能下降（甚至在某些架构上崩溃）。

对齐规则（x86_64）：

  • 每个类型必须对齐到它大小的倍数
  • 结构体整体对齐到最大成员对齐要求
  • char 对齐1，short 对齐2，int 对齐4，double 对齐8

优化建议： 将大成员放在前面，小成员放在后面，减少填充。

22.2 CPU缓存层级与性能

现代CPU有L1（32KB）、L2（256KB）、L3（共享，几MB）缓存，访问速度差距巨大：

存储层次 访问延迟（约） 大小
寄存器 < 1ns 几十个
L1缓存 ~1ns 32KB
L2缓存 ~3ns 256KB
L3缓存 ~10ns 几MB
内存 ~100ns 几GB

缓存行（Cache Line）： 通常是64字节，CPU以缓存行为单位加载数据。

22.3 缓存友好的代码示例

反例：跳跃访问（缓存不友好）

性能差异： 按行访问通常比按列访问快 5-10倍。

22.4 缓存行伪共享（False Sharing）

多线程中，不同线程修改同一缓存行中的不同变量，导致缓存行频繁失效。

性能差异： 伪共享可能导致 10倍以上的性能损失。

22.5 使用 restrict 关键字优化

restrict 告诉编译器指针不会与其他指针重叠，允许更激进的优化。

注意： restrict 是C99标准，仅在C++中不支持（C++用 __restrict）。
